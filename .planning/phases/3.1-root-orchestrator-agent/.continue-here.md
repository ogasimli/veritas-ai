## Architecture Vision

Root orchestrator coordinates all validation agents in parallel:
```
audit_orchestrator (ParallelAgent)
├── numeric_validation (SequentialAgent) - Phase 3, exists now
├── logic_consistency (future Phase 4)
├── disclosure_compliance (future Phase 5)
└── external_signal (future Phase 6)
```

The numeric_validation agent is a 3-stage sequential pipeline:
- Extractor: Identifies FSLIs
- FanOutVerifier: Parallel verification per FSLI
- Reviewer: Filters, re-verifies, assigns severity

## Key Technical Details

- Use `git mv` to preserve history when moving numeric_validation
- ParallelAgent runs sub-agents concurrently and aggregates their session states
- DocumentProcessor needs updated findings extraction: `final_state.get("numeric_validation", {}).get("reviewer_output", {}).get("findings", [])`
- All sub-agents return findings that get saved to database via DocumentProcessor

## Why This Matters

This phase establishes the foundation for multi-agent coordination. Once complete, phases 4-6 can simply add their agents as new sub-agents under the orchestrator without touching the core structure.
